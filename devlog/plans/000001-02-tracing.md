# Phase 0 — Step 2: tracing wiring

## Thinking

The design doc commits to `tracing` to `~/.local/state/argus/argus.log` (Phase 0, line 2). This plan turns that one line into wired-up code: where init lives, what format we log in, where the file lives, and how individual binaries call into it.

### Where init lives

The init code goes in **`argus-core`**, exposed as a small module. Reason: three of the five crates have a `main` (`argus-daemon`, `argus-tui`, `argus-mcp`) and will each want logging at some point. Putting the shared setup in `argus-core` means each binary calls one function rather than copying ~30 lines of subscriber-builder boilerplate.

**Only `argus-daemon` wires it this step.** TUI and MCP have different output constraints (TUI mustn't write to stdout/stderr because that scrambles the terminal; MCP stdio mode uses stdout as the JSON-RPC channel). They get wired in their respective phases, with their own `init_*` variants if the daemon variant doesn't fit.

### Format and layers

- **File layer:** JSON, append mode. Greppable, structured-tool friendly, no rotation for now.
- **Console layer:** pretty format on stderr, **only when stderr is a TTY**. So `argus-daemon` started from a shell shows logs in-terminal, but a headless / piped invocation doesn't double-print.
- **Filter:** `RUST_LOG` env var via `EnvFilter`, default `info`. Standard tracing convention; no surprises.

### Path

Literal to the design doc: `$HOME/.local/state/argus/argus.log`. Same path on Linux and macOS (the design doc didn't switch to `~/Library/Application Support/...` on macOS, so we don't either). Resolve `$HOME` with `std::env::var("HOME")`. Create parent dir if missing. No `dirs`/`directories` crate dependency — YAGNI.

### Non-blocking appender

Use `tracing_appender::non_blocking` so log writes don't block the calling thread (matters once PTY I/O and transport handlers come online). Returns a `WorkerGuard` that **must be kept alive in `main`** — drop equals flush + thread shutdown. Convention: `let _guard = argus_core::logging::init(...)?;` at the top of `main`.

### Open questions / deferred

- **Log rotation.** Skipped for now. Single append-only file. Per-session logs (design doc §7) have separate rotation requirements and aren't this file.
- **`ARGUS_LOG_FILE` env override.** Out of scope; add when a test or sandbox actually needs it.
- **Error type for `argus-core`.** Returning `anyhow::Result` for now. Custom error enum is overkill until we have failure modes to discriminate.

## Plan

1. **Workspace dependencies** in root `Cargo.toml` (pinned once, inherited by crates):
   - `tracing = "0.1"`
   - `tracing-subscriber = { version = "0.3", features = ["env-filter", "fmt", "json"] }`
   - `tracing-appender = "0.2"`
   - `anyhow = "1"`

2. **`argus-core`**:
   - `Cargo.toml`: declare the four deps from step 1 with `.workspace = true`.
   - `src/lib.rs`: `pub mod logging;` (only that; no placeholder code).
   - `src/logging.rs`:
     - `pub struct Config { pub log_file: PathBuf, pub console: bool, pub default_filter: &'static str }`
     - `pub fn default_config() -> Config` — `$HOME/.local/state/argus/argus.log`, `console = std::io::stderr().is_terminal()`, filter `"info"`.
     - `pub fn init(config: Config) -> anyhow::Result<WorkerGuard>` — creates parent dir, opens appender (`tracing_appender::rolling::never(parent, file)`), wraps in non-blocking, layers a JSON file layer with the env filter, and conditionally adds a pretty stderr layer. Returns the WorkerGuard.

3. **`argus-daemon`**:
   - `Cargo.toml`: deps `argus-core = { path = "../argus-core" }`, `tracing.workspace = true`, `anyhow.workspace = true`.
   - `src/main.rs`:
     ```rust
     fn main() -> anyhow::Result<()> {
         let _guard = argus_core::logging::init(argus_core::logging::default_config())?;
         tracing::info!("argus-daemon starting");
         Ok(())
     }
     ```

4. **Verify**:
   - `cargo fmt --all`
   - `cargo clippy --all-targets --all-features -- -D warnings` (clean)
   - `cargo check --workspace`
   - `cargo run -p argus-daemon` — process exits zero; `~/.local/state/argus/argus.log` exists and contains one JSON line with `"argus-daemon starting"`.
   - `RUST_LOG=debug cargo run -p argus-daemon` — exits zero (filter parse path exercised).

5. **Update branch devlog** — flip tracing checkbox, add What Changed entries, update Commits (previous HEAD → real hash, new HEAD entry).

6. **Commit** `chore: wire tracing to ~/.local/state/argus/argus.log` and push to keep PR #1 current.

## Out of scope

- TUI and MCP tracing — wired in Phase 3 / Phase 5 plans with their own variants.
- Log rotation — until we measure file growth or need it.
- Env-var overrides for the log path — add when a test needs it.

## Amendment — 2026-05-11

The design roadmap now treats daemon session semantics as an earlier prerequisite than the old phase numbering implied. TUI and MCP logging should still use variants that avoid corrupting their transport surfaces, but their phase labels changed:

- TUI logging belongs with the local TUI work after daemon session state and local IPC are stable.
- MCP logging belongs with the local MCP server phase and must never write to stdout in stdio mode.
- Any tracing added around session control should include input lease acquisition, takeover, release, and rejected injection events so multi-client behavior can be debugged from logs.
